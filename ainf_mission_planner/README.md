# ainf_mission_planner

`ainf_mission_planner` is a RoboComp GUI client for the `Navigator` interface (served by `ainf_slamo`).
It visualizes the current map/layout, object polygons, robot pose, and planned paths, and provides basic mission control actions from the UI.

## What this component does

- Connects to `Navigator` proxy.
- Requests map/layout once at startup with `getLayout()`.
- Draws:
    - room outline (magenta, thick),
    - objects (different color per object, filled),
    - object labels.
- Updates robot pose in `compute()` using `getRobotPose()`.
- Allows Shift+Right click on the map to request a path (`getPath(source, target, safety)`) and draw it.
- Provides control buttons for path following:
    - **Start Follow**: calls `gotoPoint(last_target)`
    - **Stop**: calls `stop()`
    - **Resume**: calls `resume()`
- Displays robot coordinates in meters/radians in the top bar.
- Saves and restores main window geometry between runs.

## Interface used

This component uses `RoboCompNavigator` (`Navigator.idsl`) and calls:

- `getLayout()`
- `getRobotPose()`
- `getPath(TPoint source, TPoint target, float safety)`
- `gotoPoint(TPoint target)`
- `stop()`
- `resume()`

## Dependencies

- RoboComp environment correctly installed (`$ROBOCOMP` set)
- Ice / IceStorm
- Qt6 (Widgets/OpenGL stack used by `AbstractGraphicViewer`)
- `abstract_graphic_viewer` class from RoboComp classes

## Configuration

Use `etc/config` (or `etc/config.toml`) and make sure `Proxies.Navigator` points to a running `ainf_slamo` (or another `Navigator` provider).

## Build

From the component root:

```bash
cmake .
make -j
```

## Run

```bash
bin/ainf_mission_planner --Ice.Config=etc/config
```

## User manual

### 1) Startup

1. Start `ainf_slamo` first.
2. Start `ainf_mission_planner`.
3. On startup, the component loads and draws the current map and objects.

### 2) UI layout

- **Top bar**:
    - `Start Follow`, `Stop`, `Resume` buttons.
    - Robot coordinates label: `x`, `y` in meters and `θ` in radians.
- **Main area**:
    - 2D viewer with room, objects, robot, and planned path.

### 3) Create and visualize a path

1. In the viewer, press **Shift + Right Click** on the destination.
2. The component requests a path from current robot pose to clicked target.
3. Returned path is drawn in orange, with start/goal markers.

### 4) Execute path following

1. After selecting a target/path, click **Start Follow**.
2. Click **Stop** to pause navigation.
3. Click **Resume** to continue.

### 5) Navigation and view tips

- Mouse interactions are provided by `AbstractGraphicViewer`.
- Robot pose is refreshed cyclically from `Navigator`.
- Window size/position is restored automatically at next launch.

## Troubleshooting

### 1) Component starts but map is not shown

- Check that `ainf_slamo` is running.
- Verify `Proxies.Navigator` in `etc/config` points to the correct endpoint.
- Inspect console warnings for `Error requesting map from Navigator`.

### 2) Robot marker does not move

- Confirm `getRobotPose()` is available and returning updated values in `ainf_slamo`.
- Check runtime logs for `Error requesting robot pose from Navigator`.
- Ensure both components use the same map/reference frame.

### 3) Shift+Right click does not produce a path

- Make sure you are using **Shift + Right Click** (not left click).
- Verify the clicked target is in reachable free space.
- Check warnings:
    - `Path request failed: ...`
    - `Path request returned an empty or single-point path`

### 4) Start Follow does nothing

- `Start Follow` requires a previously selected target (via Shift+Right click).
- If no target exists, select one first and retry.
- Check `Navigator` side behavior for `gotoPoint` acceptance/rejection.

### 5) Stop/Resume buttons do not affect motion

- Verify `stop()` and `resume()` are implemented/enabled in your `Navigator` server.
- Check that no higher-level controller is overriding robot commands.

### 6) Text or graphics look odd after changing viewer code

- The viewer uses a flipped Y axis; text needs compensation transforms.
- Rebuild after UI/view changes:

```bash
make -j
```
